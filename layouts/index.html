Go-Fed provides two libraries and a tool to start working with ActivityStreams and ActivityPub data within your project.

What is ActivityPub? What is Go-Fed?

ActivityPub is a protocol that different social applications use to communicate with each other. Whether you want to create a new social app or retro-fit social functionality into an existing one, ActivityPub lets everyone use each other's network effect by pitching in together! This wider network-of-networks is known as the Fediverse. Users of many different software -- such as Mastodon, Pleroma, Pixelfed, Peertube, WriteFreely, Read.as, and others -- can talk, follow, share, and like each other's content regardless of the software they are using.

More about ActivityPub

Go-Fed is unlike most other software in the Fediverse. Instead of providing a monolithic end-product, it is the first standalone ActivityPub library, and a pioneering solution in Go. Go-Fed is designed for those who have no prior experience with ActivityPub, ActivityStreams, JSON-LD, or RDF. This library has been successfully used to federate since May 17, 2019, and it has reached version 1.0.0 as of July 9, 2020.

Using Go-Fed

The following is a list of software which implements the Go-Fed libraries:

  • WriteFreely
    A simple, open-source, privacy-focused blogging platform. WriteFreely is the software powering the write.as service.
  • Read.as
    A long-form reader built on open protocols. Read.as powers the read.write.as service.
  • apcore
    A generic ActivityPub server framework in Go. apcore is a reference implementation of the Go-Fed suite of libraries.
  • AP Test Suite
    The unofficial ActivityPub test suite, hosted at test.activitypub.dev.

If you are using Go-Fed in your application and want to be added to this list, contact @cj@mastodon.technology on the fediverse, or email cjslep@gmail.com. You can also try writing your own software that uses Go-Fed! Go-Fed is used by various personal blogs, tinkerers, and people that want ponies.

"Go-Fed: Past, Present and Future" is a talk that was given at ActivityPub Conf 2020. The talk covers the principles behind Go-Fed, describes Go-Fed's architectural decisions such as being split into the activity/streams and activity/pub libraries, examines the technical reasons for the design of the go-fed/activity library, and explores future development potential. You can find more talks on conf.tube, or you can read the Go-Fed reference material to learn more about the internals of Go-Fed.
